Device Tree

Способ подключения устройств Antilatency друг к другу и объединения их в одну сеть.
Все устройства Antilatency, подключённые к одному Host, образуют сеть в виде дерева устройств. Корневой точкой является Host, а остальные устройства образуют узлы, или Node. Каждый узел этого дерева может выполнять Task. Дерево устройств формирует иерархию подключения устройств и порядок передачи данных на Host.
Дерево устройств отображается в приложении AntilatencyService во вкладке Device Network. Здесь вы можете управлять свойствами любого из подключённых устройств.
  • Чтобы настроить HMD Radio Socket, Socket USB и Pico G2 Socket подключите их напрямую через USB.
  • Tag и Bracer, нужно подключить их через HMD Radio Socket.
  • Alt можно подключить через любой совместимый Socket.
Подробнее о конфигурации устройств и работе с деревом устройств читайте тут.
Для обмена данными устройства Antilatency могут использовать проприетарный радиопротокол. В зависимости от роли в процессе обмена данными, все устройства можно разделить на две группы: клиенты и точки доступа. Подробнее читайте тут: Antilatency Radio Protocol.
Рассмотрим дерево устройств на примере полного комплекта устройств для одного пользователя:
  • HMD Radio Socket и трекер Alt - для отслеживания головы;
  • два Bracer и два трекера Alt - для отслеживания рук;
  • два Tag и два трекера Alt - для отслеживания ног.
HMD Radio Socket, подключённый к шлему виртуальной реальности по USB, выполняет роль точки доступа. А Tag и Bracer выступают как клиенты. Вместе устройства образуют сеть, с помощью которой можно отслеживать положение тела пользователя на основе знаний о позиции пяти точек.
