Environments

Инструмент AntilatencyService для управления зонами трекинга. С его помощью можно не только создать или загрузить зону, но также посмотреть её характеристики, проверить качество трекинга и настроить параметры, используя редактор зон трекинга.
Подробнее о том, как собрать зону трекинга:

Используя опцию "Set as default"...
... или долгим нажатием.

Ссылка на зону трекинга

Данные зоны трекинга можно отправить другому пользователю в виде ссылки, которая имеет такой вид:
http://www.antilatency.com/antilatencyservice/environment?data=AAVSaWdpZE8BBnllbGxvdwgIuFOJP9xGoD6vjmO9mpmZPgAAAAAAAAAAAJqZGT8NBQAEAwAGAwYCBAIDAAUHBgQDAwUAAAIHBgcGBwYHBwIHAgcGAgMDdzk&name=DevKitX&setAsDefault=true&silent=true

Ссылка может содержать несколько параметров, из которых два - data и name - обязательные.
data - данные зоны трекинга;
name - её название (можно изменить в AntilatencyService);
setAsDefault - автоматически назначает добавленную зону трекинга зоной по умолчанию;
silent - актуально для Android; позволяет добавить зону трекинга нажатием по ссылке, не открывая окно AntilatencyService.

Используя опцию "Share", можно скопировать и отправить другому пользователю ссылку на зону трекинга.

Добавить новый проект по ссылке можно при помощи опции "From link".
Если вы заранее скопировали ссылку, то форма добавления зоны трекинга будет автоматически заполнена данными из буфера обмена.

Чтобы добавить зону трекинга на Android устройстве, можно использовать опцию "From Link" или нажать непосредственно на ссылку. Тогда откроется уже заполненная форма добавления новой зоны в AntilatencyService. Если в ссылке присутствует параметр silent, то зона трекинга будет добавлена автоматически без вызова окна приложения.
