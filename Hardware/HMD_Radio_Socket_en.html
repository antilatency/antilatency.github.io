HMD Radio Socket

The HMD Radio Socket is a wireless Socket that supports both data reception from wireless devices via the Antilatency Radio Protocol and tracking data transmission when placed on the objects being tracked.

A universal data receiver via a radio protocol

When the Antilatency Radio Protocol is running, as a rule, the HMD Radio Socket serves as a data receiver from transmitters. In this case the USB type-C port is used to power the HMD Radio Socket and send data to the Host via USB.
For other modes of operation, please see HMD Radio Socket.

Operating as a data transmitter

Using the “Mode” property, you can configure the HMD Radio Socket to work as a data transmitter via a radio protocol. In this case the HMD Radio Socket is powered by an external power bank.

External power source

You can power the HMD Radio Socket from an external power bank via the USB type-C port. In this case the HMD Radio Socket functions as a transmitter, similarly to the Tag

Extension board support

The HMD Radio Socket supports the Antilatency Extension Board. The Antilatency Extension Board connects to the HMD Radio Socket through the USB type-C port to transmit data about external triggers and response control (vibration, etc.). The HMD Radio Socket sends these data, along with tracking data, to its receiver.
For more details, please see Antilatency Hardware Extension Interface.

Modular design

The HMD Radio Socket can be used to track objects instead of the Tag if the tracked object has its own battery or an external battery can be fitted inside or outside the object's casing. In this capacity the HMD Radio Socket has smaller dimensions than the Tag and gets its power from an external power bank.
The HMD Radio Socket can be attached with double-sided adhesive tape or using its mounting holes for a secure fit (TODO for more information, please see the file with a model of the casing).
