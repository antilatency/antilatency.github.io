Режим работы HMD Socket

HMD Radio Socket работает в одном из двух режимов: wired или wireless. Используя системное приложение AntilatencyService, вы можете настроить режим работы любого подключённого устройства.
Начиная с версии прошивки 1.3.0, HMD Radio Socket поддерживает два режима работы:
  • wired, или radio master - устройство подключает к себе другие беспроводные Socket, а данные передаёт через USB)
  • wireless, или radio slave - устройство подключается к проводному HMD Radio Socket, а данные передаёт по радиоканалу)

Вы можете изменить режим работы HMD Radio Socket во вкладке Device Network служебного приложения AntilatencyService.
Для этого установите для свойства Mode одно из двух значений: UsbRadioSocket или RadioSocket. Каждое из них обладает независимым набором свойств для конфигурации. Пользовательские свойства, к примеру, Tag, - общие для всех режимов работы.
Обновление прошивки доступно только в режиме UsbRadioSocket.

UsbRadioSocket (wired)

Уникальные свойства:
  • RadioChannel - по умолчанию установлено значение -1. Приёмник в случайном порядке выбирает первый свободный радиоканал из доступных. Подробнее о передаче данных по радиоканалу см. Antilatency Radio Protocol
  • ConnLimit - максимальное количество передатчиков, которые могу быть подключены к этому приёмнику. Если значение свойства 0, то радиопередача через это устройство будет недоступна.
Мы рекомендуем указывать точное количество устройств, которое вы планируете подключить. Если вы пропишете большее значение, то часть траффика будет тратиться на поиск новых устройств.

RadioSocket (wireless)

С версии прошивки 5.0.0 HMD Radio Socket в wireless-режиме отображается в Device Tree под именем AltHmdRadioSocketShadow. Для этого устройство необходимо подключить через USB.
При этом вам доступны все возможности для настройки параметров wireless режима: можно установить маску каналов, прописать серийный номер radio master или изменить режим работы.

Уникальные свойства:
  • MasterSN - свойство, содержащее серийный номер приёмника, к которому будет подключаться конкретный передатчик. Задать MasterSN можно в AntilatencyService или при помощи кнопки на устройстве.
    Подробнее читайте тут: Конфигурация беспроводных устройств.
  • ChannelsMask - свойство задаёт маску каналов, по которой передатчик будет искать приёмник. Это строка длиной в 141 символ, состоящая из 0 и 1, где 1 означает, что соответствующий канал будет использован при поиске приемника, а 0 - что канал будет проигнорирован. Первый символ в строке отвечает за последний, 140-й канал. Маска для каналов по умолчанию выглядит следующим образом: 000000000000000000001000001000000000000000000000100000000000000000000000001000000000000000000000001000000000000000000000000000000000000000000 Положение символов 1 в ней соответствует списку каналов, доступных по умолчанию.
    Вы можете прописать маску через 0 и 1 или использовать альтернативный формат записи:
    • full - для поиска активны все каналы
    • default - для поиска активны только 5 каналов по умолчанию
    • N - для поиска активен только канал N. Например, для 140 будет установлена маска 100....000
