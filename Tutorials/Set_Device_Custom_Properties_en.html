Назначение кастомных свойств для устройств Antilatency

Инструкция по назначению кастомных свойств для устройств на примере назначения свойства Tag для Tag
У каждого устройства Antilatency есть ряд Device Properties. К примеру, кастомное свойство Tag позволяет определить конкретное устройство среди прочих, что несомненно облегчает процесс разработки. В этом туториале мы рассмотрим назначение кастомных свойств на примере свойства Tag у Tag.

Установка

1. Подключите HMD Radio Socket к компьютеру, или любому другому хосту.
2. Подключите Tag к HMD Radio Socket. Подробнее о подключении можно узнать здесь: Wireless Devices Configuration
3. Откройте Antilatency Service, вкладку Device Network
4. Выберите Tag в Device Tree. Во вкладке справа назначьте нужное свойство Tag (в нашем случае Gun). После ввода нового свойства слева от него появится значок жёлтой ручки.
5. Теперь отправим новое свойство на устройство. Для этого воспользуйтесь синей кнопкой в правом нижнем углу окна. Выберите Send settings to device
После нажатия кнопки данные запишутся на устройство и пропадёт значок жёлтой ручки. Вы также можете воспользоаться Reload settings from device, чтобы заново загрузить данные с устройства.
