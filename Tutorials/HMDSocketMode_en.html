HMD Socket Modes

The HMD Radio Socket has two modes: wired and wireless. Use AntilatencyService to configure the mode for any attached Socket. Each mode defines the device properties in accordance with its functions.
Since firmware version 1.3.0, HMDRadioSocket has two modes:
  • wired, or radio master - any wireless Socket can connect to this one; radio master transfer the data via USB;
  • wireless, or radio slave - such a device can connect to the HMDRadioSocket device and transfer the data via a radio channel.

If you need to change the mode use the Device Network tab in AntilatencyService. For the Mode property use one of the following values:
  • UsbRadioSocket (wired)
  • RadioSocket (wireless)
You can update the Socket firmware ONLY in UsbRadioSocket mode.

The device in each mode has an independent set of properties. The custom properties, for example, Tag, are the same for all the modes.
Please, read here to learn more about the device's properties: Wireless Devices Configuration.

UsbRadioSocket

Special properties

  • RadioChannel - by default, the receiver chooses one random radio channel from the five available. Please, read here to learn more about the radio protocol: Antilatency Radio Protocol
  • ConnLimit - the max number of Sockets you can connect to this receiver.
We highly recommend setting the exact amount of the devices you are going to connect. This way you avoid you wastine traffic searching for new devices.

RadioSocket

Since firmware version 5.0.0 the wireless HMD Radio Socket appears in the Device Tree as an additional node AltHmdRadioSocketShadow. But firstly, you need to connect the device via USB.
You can also configure its properties: set the channel mask, the MasterSN or change the Socket mode.
If you connect one Socket using both USB and radio connections, it will be displayed in the Device Tree twice under different names.

Special properties

  • MasterSN - defines the serial number of the master device (receiver);
  • ChannelsMask - defines the channel mask used to find the receiver to connect to.
